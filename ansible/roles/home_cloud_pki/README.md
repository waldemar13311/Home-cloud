# Role: Home Cloud PKI

Роль для развертывания и управления внутренней инфраструктурой открытых ключей (PKI) в домашнем облаке.

## Что делает эта роль
1.  **Локально (на управляющей машине):**
    * Создает структуру директорий для хранения секретов.
    * Генерирует корневой сертификат (Root CA), если он отсутствует.
    * Генерирует приватные ключи, CSR и выпускает сертификаты для каждого хоста из инвентаря.
2.  **Удаленно (на целевых серверах):**
    * Добавляет корневой сертификат в доверенное хранилище ОС (`ca-certificates`).
    * Копирует персональные сертификаты и ключи в целевую директорию (например, `/etc/ssl/internal`).

## Требования
* Установленная коллекция `community.crypto` на локальной машине.
* Наличие `python3-pyopenssl` или `cryptography` на локальной машине.
* Целевые хосты должны иметь доступ к `sudo` (для обновления доверенных сертификатов).

## Переменные роли

Основные переменные находятся в `defaults/main.yml`:

| Переменная | Описание | Значение по умолчанию |
|------------|----------|-----------------------|
| `home_cloud_pki_local_secrets_dir` | Путь на Mac/PC для хранения ключей | `../../secrets` |
| `home_cloud_pki_target_secrets_dir` | Путь на серверах для деплоя сертификатов | `/etc/ssl/internal` |
| `home_cloud_pki_ca_cert_days` | Срок действия Root CA | `+3650d` |
| `home_cloud_pki_leaf_cert_days` | Срок действия сертификатов серверов | `+1825d` |
| `home_cloud_pki_trusted_store_path` | Путь к системному хранилищу CA | `/usr/local/share/ca-certificates/` |

## Использование в плейбуке

Пример запуска роли для всех хостов:

```yaml
---
- name: Init Home Cloud PKI
  hosts: all
  gather_facts: true
  roles:
    - role: home_cloud_pki
      vars:
        home_cloud_pki_local_secrets_dir: "./secrets"
```

## Структура секретов после запуска

После выполнения роли, на локальной машине структура будет выглядеть так:

```bash
secrets/
├── root_ca.crt
├── root_ca.csr
├── root_ca.key
├── master.home/
│  ├── server.crt
│  ├── server.csr
│  └── server.key
└── nginx.home/
   ├── server.crt
   ├── server.csr
   └── server.key
```
